# Kế hoạch: Nâng cấp Sidebar hỗ trợ hiển thị tài liệu đa cấp (Nested Docs)

Sau khi điều tra mã nguồn hiện tại, tôi xác nhận hệ thống **ĐANG THIẾU** tính năng hiển thị tài liệu nhiều tầng (Nested Sub-docs).
- **Hiện trạng**: Cấu trúc dữ liệu (`NavItem`) và giao diện (`Sidebar`) chỉ hỗ trợ danh sách phẳng 2 cấp (Nhóm -> Bài viết). Không thể hiển thị dạng cây (Cha -> Con -> Cháu) như Outline.
- **Giải pháp**: Cần nâng cấp cả cấu trúc dữ liệu và giao diện Sidebar để hỗ trợ đệ quy (Recursive Rendering).

Dưới đây là kế hoạch thực hiện chi tiết:

## 1. Nâng cấp Cấu trúc Dữ liệu (Data Structure)
Sửa đổi file `lib/docs/nav.ts` để hỗ trợ cấu trúc đệ quy.

```typescript
// Trước
export interface NavItem {
  title: string
  href: string
  icon?: React.ComponentType
}

// Sau (Đề xuất)
export interface NavItem {
  title: string
  href?: string // Có thể undefined nếu chỉ là folder
  icon?: React.ComponentType
  items?: NavItem[] // Hỗ trợ lồng nhau vô hạn
  collapsible?: boolean // Tùy chọn đóng mở
}
```

## 2. Tái cấu trúc Sidebar Component (UI/UX)
Sử dụng component `Collapsible` có sẵn trong dự án (`components/ui/collapsible.tsx`) để xây dựng menu cây.

### Các công việc cụ thể:
1.  **Tạo component `SidebarItem` mới**:
    *   Có khả năng tự gọi lại chính nó (recursive) nếu có `items` con.
    *   Tích hợp logic đóng/mở (Expand/Collapse).
    *   Thêm thụt lề (indentation) tự động dựa trên độ sâu (level).
2.  **Xử lý Active State thông minh**:
    *   Khi người dùng vào một bài viết con (ví dụ: `/docs/a/b/c`), Sidebar phải tự động mở rộng các thư mục cha (`a` và `b`) và highlight bài viết `c`.
3.  **Cập nhật file `components/layout/sidebar.tsx`**:
    *   Thay thế vòng lặp `map` đơn giản hiện tại bằng component đệ quy mới.

## 3. Cập nhật Dữ liệu Mẫu (Mock Data)
Thêm dữ liệu mẫu 3-4 cấp vào `lib/docs/nav.ts` để kiểm thử hiển thị ngay lập tức (trước khi có dữ liệu thật từ Outline).

---
**Lưu ý**: Việc này hoàn toàn tương thích với **Phương án 2 (Outline Integration)** mà bạn đã chọn. API của Outline trả về cấu trúc cây (Collection -> Document -> Nested Document), nên việc nâng cấp Frontend này là bắt buộc để hiển thị đúng dữ liệu từ Outline sau này.
