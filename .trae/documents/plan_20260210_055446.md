Tôi sẽ thực hiện việc bổ sung tính năng đánh giá tài liệu 2 bước như sau:

1. **Tạo component mới** **`DocsRating`** (`components/docs/docs-rating.tsx`):

   * **Logic:** Sử dụng state để quản lý quy trình 2 bước:

     * **Bước 1:** Hỏi "Do you find this doc helpful?" với 3 lựa chọn: Bad (Buồn), Normal (Bình thường), Useful (Vui).

     * **Bước 2:** Hỏi "Do you find it easily to understand?" với 3 lựa chọn: Not really (Buồn), Normal (Bình thường), Easy (Vui).

     * **Hoàn tất:** Hiển thị thông báo cảm ơn sau khi hoàn thành.

   * **Giao diện:** Sử dụng các icon từ thư viện `lucide-react` (Frown, Meh, Smile) để biểu thị cảm xúc. Thiết kế sẽ sử dụng các component UI có sẵn như `Button` và `Card` (nếu có) để đồng bộ với giao diện hiện tại.

   * **Dữ liệu:** Tạm thời lưu trạng thái ở phía client (console log kết quả), chuẩn bị cấu trúc dữ liệu để sau này tích hợp Supabase dễ dàng.

2. **Tích hợp vào** **`DocContent`** (`components/docs/doc-content.tsx`):

   * Import component `DocsRating`.

   * Đặt component này ở cuối nội dung bài viết, ngay trước phần điều hướng trang (Pager), để người dùng dễ dàng đánh giá sau khi đọc xong.

**Cấu trúc dữ liệu dự kiến (để lưu DB sau này):**

```typescript
interface DocRating {
  docSlug: string;
  isHelpful: 'bad' | 'normal' | 'useful';
  isEasyToUnderstand: 'not_really' | 'normal' | 'easy';
  createdAt: string;
}
```
